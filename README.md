# Atividade – Tratamento de Erros e Avaliação de Soluções

## 📌 Introdução

Durante o desenvolvimento do projeto, percebemos que alguns erros acabavam atrapalhando o funcionamento do sistema. Muitos deles eram simples, como erros de sintaxe, mas outros estavam relacionados à lógica do programa e ao tratamento de dados inseridos pelo usuário.

Na aula sobre tratamento de erros, vimos a importância de identificar essas falhas para evitar problemas maiores durante a execução do sistema. Além disso, entendemos que um software precisa estar preparado para lidar com situações inesperadas sem parar de funcionar completamente.

## 🎯 Objetivos

- ✅ Identificar diferentes tipos de erros em código Python (sintaxe, lógica e execução)
- ✅ Aplicar técnicas de tratamento de exceções
- ✅ Melhorar a qualidade e estabilidade do código
- ✅ Compreender a importância da validação de dados
- ✅ Avaliar soluções implementadas

## 📂 Estrutura da Atividade

Este repositório contém:

| Arquivo | Descrição |
|---------|----------|
| `Erros_Identificados.md` | Registro detalhado dos erros encontrados durante o desenvolvimento |
| `Projeto_Corrigido.md` | Versão corrigida do código com justificativas para cada correção |
| `Avaliacao.md` | Reflexão sobre eficiência, clareza e escalabilidade da solução final |

## 🔍 Resumo das Melhorias

Após a análise dos erros, implementamos:

- ✨ Uso de tratamento de exceções (`try/except`)
- 🔐 Validação dos dados inseridos pelo usuário
- 📦 Melhor organização do código
- 🔀 Separação das funções do sistema
- 💬 Mensagens de erro mais claras
- 🧪 Testes em diferentes situações

## 💡 Principais Aprendizados

1. **Erros de sintaxe** são mais fáceis de identificar, pois impedem a execução
2. **Erros de lógica** são mais perigosos, pois o código executa mas com resultados errados
3. **Erros de execução** afetam a estabilidade e exigem validação de entrada
4. **Tratamento de exceções** é fundamental para criar sistemas robustos

## 📊 Impacto das Melhorias

| Aspecto | Antes | Depois |
|---------|-------|--------|
| **Estabilidade** | Múltiplos travamentos | Sistema estável |
| **Clareza** | Código desorganizado | Bem estruturado |
| **Validação** | Sem verificação | Validação completa |
| **Manutenibilidade** | Difícil | Fácil de manter |
